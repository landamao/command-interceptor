# 指令拦截器

一个 AstrBot 插件，用于拦截指令消息，防止 LLM 被误唤醒。

## 功能特性

- **自动检测指令**：自动遍历所有已注册的插件指令（包括别名），无需手动维护
- **拦截指令唤醒**：当用户发送 `/xxx` 形式的指令时，阻止 LLM 将其当作普通聊天处理
- **拦截前缀唤醒**：可选拦截以唤醒前缀（如 `!`、`#` 等）开头的消息
- **拼音排序**：中文指令按拼音排序，英文指令按字母排序，方便查找

## 配置说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 名字 | 字符串 | `小助手` | 合并转发消息中机器人显示的名称 |
| 额外指令 | 列表 | `[]` | 除自动检测外，额外需要拦截的指令 |
| 禁用指令唤醒 | 布尔 | `true` | 拦截已注册指令，阻止唤醒 LLM |
| 禁用前缀唤醒 | 布尔 | `false` | 拦截带唤醒前缀的消息 |

## 指令列表

### /所有指令

以合并转发形式展示所有已注册的指令列表，避免刷屏。

### /验证指令 `<指令名>`

检查某个指令是否存在于已注册列表中。

```
/验证指令 帮助
```

✅ 该指令「帮助」在所有指令中

### /添加额外指令 `<指令名>`

别名：`/添加指令`、`/额外指令`

仅管理员可用。快速添加额外拦截指令，支持逗号分隔批量添加。添加后自动保存配置并刷新指令缓存。

```
/添加额外指令 天气,签到,帮助
```

✅ 已添加指令：天气，签到，帮助

如果指令已存在则跳过，并提示哪些指令重复。

### /刷新指令

仅管理员可用。当新插件加载或指令变更后，使用此命令重新扫描所有已注册指令并更新缓存，返回新增和减少的指令详情。

```
/刷新指令
```

✅ 已刷新所有指令
新增：天气, 签到
减少：旧功能

## 工作原理

插件通过 `@filter.on_llm_request` 钩子（最高优先级 `sys.maxsize`）在 LLM 处理消息前进行拦截：

1. 将用户消息的第一个词与所有已注册指令进行匹配
2. 如果命中且「禁用指令唤醒」开启，则直接 `stop_event()` 阻止后续处理
3. 如果消息以唤醒前缀开头且「禁用前缀唤醒」开启，同样拦截
